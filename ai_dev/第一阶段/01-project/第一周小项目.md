🎯 项目目标：智能文档摘要器

功能描述：
读取当前目录下名为 source_text.txt 的文件。
将内容发送给大模型，要求它生成一段简短的摘要。
将大模型返回的摘要保存到 summary_result.txt 中。

💻 代码实现

这段代码使用了 f-string 来构建 Prompt，这是最基础的模板化思想。

import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI

加载环境变量
load_dotenv()

初始化模型 (使用 DeepSeek 配置)
llm = ChatOpenAI(
    model="deepseek-chat",
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com"
)

读取本地文件
input_file = "source_text.txt"
output_file = "summary_result.txt"

try:
    print(f"📖 正在读取文件: {input_file} ...")
    with open(input_file, "r", encoding="utf-8") as f:
        content = f.read()
    
    # 简单的防御性编程：如果文件是空的就不跑了
    if not content.strip():
        print("❌ 文件内容为空，请检查 source_text.txt")
    else:
        # 4. 构建 Prompt (提示词工程)
        # 这里的 f-string 就是最原始的模板，{content} 是占位符
        prompt_text = f"""
        请阅读以下文本，并用简练的语言总结其核心内容（100字以内）：

        {content}

        """

        print("🤖 正在调用大模型进行摘要...")
        # 5. 调用模型
        response = llm.invoke(prompt_text)
        summary = response.content

        # 6. 保存结果
        print(f"💾 正在保存结果到: {output_file} ...")
        with open(output_file, "w", encoding="utf-8") as f:
            f.write(summary)
            
        print("✅ 任务完成！摘要如下：")
        print("-" * 30)
        print(summary)
        print("-" * 30)

except FileNotFoundError:
    print(f"❌ 错误：找不到文件 {input_file}，请确保它和脚本在同一目录下。")
except Exception as e:
    print(f"❌ 发生未知错误: {e}")

🔍 代码关键点解析

with open(..., encoding="utf-8"):
    这是 Python 处理文件的标准姿势。with 语句能保证文件用完后自动关闭，不会占用资源。
    重点：一定要加 encoding="utf-8"，否则在中文环境下容易报错或乱码。

Prompt 的结构：
    注意看 prompt_text，我用了 --- 分隔符。
    这是一种技巧，告诉大模型：“分隔线中间的是数据，上面的是指令”。这能防止如果你的文本里包含类似指令的话，模型产生混淆。

异常处理 (try...except)：
    做工程化开发时，如果文件不存在，程序直接崩溃是很不友好的。用 try...except 捕获 FileNotFoundError 可以给出更清晰的提示。
