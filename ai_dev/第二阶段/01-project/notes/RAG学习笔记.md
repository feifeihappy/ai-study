# RAG 学习笔记

## RAG 是什么

RAG（Retrieval Augmented Generation，检索增强生成）是一种让大模型在回答问题时，**先检索外部知识库，再基于检索结果生成答案**的技术。

## 为什么需要 RAG

大模型有两大局限：
1. **知识截止日期**：训练数据停在某个时间点，不知道最新信息
2. **幻觉问题**：对于不知道的事，模型可能编造答案

RAG 解决了这两个问题：检索到的文档是最新的，且基于真实文档回答，大幅减少幻觉。

## RAG 的两阶段

### 离线阶段（索引）
1. 加载文档（PDF/Markdown/网页）
2. 文本分割（RecursiveCharacterTextSplitter）
3. 向量化（Embedding 模型）
4. 存储到向量数据库（ChromaDB/FAISS/Milvus）

### 在线阶段（检索 + 生成）
1. 用户提问
2. 将问题向量化
3. 在向量数据库中检索最相关文档
4. 将检索结果作为上下文，和问题一起发给大模型
5. 大模型生成答案

## 关键调参

- **chunk_size**：500~1000 字符适合中文。太小信息碎片化，太大检索精度下降
- **chunk_overlap**：设为 chunk_size 的 10%~20%，保持上下文连贯
- **top_k**：检索返回 3~5 个 chunk，太多会引入噪音
